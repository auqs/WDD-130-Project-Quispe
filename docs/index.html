Passwords should never be stored into plain text

Passwords should never be stored into plain text. Passwords are important authorization for the user. When a password is stored into plaintext, bad actors have the opportunity to exploit the website and gain credentials. Passwords should be encrypted or hashed by the site's security system, protecting users from malicious actors.

What is hashing?

Hashing is a protection to prevent from a malicious user seeing your password in plaintext. The hashing corresponds to the users password and protects users from having their passwords be in plaintext form.

Why it matters

Hashing is an important factor to secure your password in plaintext. When you create a password, an algrothimic function blends your password with a salt, a uniquely identified string corresponding to your final hash, the result of what your password is provided. This prevents a user to reverse engineer a hash without a salt, and find your password.

Best practice

Having long and complex passwords is a good pratice. Since hashed passwords can only be cracked through credential stuffing or random dictionary attacks. Bad actors would have to guess your password manually since reverse engineering a salted password is not optimal for threat actors.

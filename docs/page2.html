How Hashing Works

Password hashing starts when the user enters their regular password. The password is then "hashed", or converted to a random string using a specific mathematic function or algorithm. The "hashed" password is a string of charachters and numbers that is meaningless by itself. However, the computer system can compare the "hashed" string against the original "hashed" string, that was created when the user first chose their password. It the new hashed string and the original hashed string match, the user is authenticated and allowed access.

Hashing Unlocked

Why is Hashing so Useful?

Cyberattacks are constant threat in the information age. If a system is compromised by a cyberattack, a hacker could potentially get every user's password, if they were unhashed. Hashing allows the system to store only the hashed string from the password, instead of the password itself. This way, if the system is compromised, The hacker only has access to the hash, which by itself is meaningless. The hacker would have to test every password, see its output hash, and match it against the stored hash, which is much more difficult.
